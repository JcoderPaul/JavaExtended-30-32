******* Удаление файла или каталога *******

Мы можем удалять файлы, каталоги или ссылки. С символическими ссылками удаляется
ссылка, а не цель ссылки. В случае каталогов каталог должен быть пустым, иначе
удаление не удастся.

Класс Files предоставляет два метода удаления:
- 1. - delete(Path) - удаляет файл или выдает исключение, если удаление не удается.
                  Например, если файл не существует, выбрасывается NoSuchFileException.

Мы можем поймать исключение, чтобы определить, почему удаление не удалось:
-----------------------------------------------------------------------------
try {
    Files.delete(path);
} catch (NoSuchFileException x) {
    System.err.format("%s: no such" + " file or directory%n", path);
} catch (DirectoryNotEmptyException x) {
    System.err.format("%s not empty%n", path);
} catch (IOException x) {
    // File permission problems are caught here.
    System.err.println(x);
}
-----------------------------------------------------------------------------

- 2. - deleteIfExists(Path) - метод также удаляет файл, но если файл не существует,
                              исключение не выбрасывается.

Такая проверка перед удалением полезна, когда у нас есть несколько потоков,
удаляющих файлы, и мы не хотим создавать исключение только потому, что один
поток сделал это первым.
